# Hermes Troubleshooting Lab

Teaches users how to diagnose Hermes when something breaks.

## Tutorial level

Advanced integrator

## What it teaches

Start with `hermes doctor`, `hermes status`, targeted logs, and the exact command that failed.

## Slash commands

- `/troubleshoot`
- `/explain-error`

## Tools

- `tutorial_explain_error`
- `tutorial_suggest_recovery_steps`
- `tutorial_classify_failure`

## Bundled skills

- `hermes-troubleshooting-lab:doctor-basics`
- `hermes-troubleshooting-lab:logs-basics`
- `hermes-troubleshooting-lab:plugin-debugging`
- `hermes-troubleshooting-lab:mcp-debugging`

## Safety

This plugin is designed for tutorial use. It does not require secrets. Runtime learner progress, when present, stays in the active profile's local data path and is not part of public packages.

Official Hermes docs and live CLI output are the source of truth if tutorial content differs.
