# Identity Neutrality

Tracked public files should be written for a generic Hermes user.

Use placeholders such as:

- `$HERMES_HOME`
- `$HOME`
- `<profile>`
- `<repo>`
- `<user>`
- `example.com`

Do not include private account identifiers, private email addresses, private
profile names, private plugin names, person-specific examples, or absolute
paths from the machine that produced the public copy.

Public skill frontmatter may retain approved authorship metadata. Validators
should distinguish approved metadata from private operational context.
