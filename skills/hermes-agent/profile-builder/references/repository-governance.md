# Repository Governance Defaults for Hermes Profile/Config Distributions

Use this reference when a profile-builder request includes publishing or maintaining a Hermes configuration repository.

## Durable defaults

- Git author identity: `<repo-author-name> <<private-term>>`.
- Commit messages: no `Co-authored-by` trailers, no AI co-authoring trailers, and no AI-attribution lines.
- New repository-author-authored skills: `version: 0.1.0`, `author: <repo-author-name>`, `license: Apache-2.0`.
- Public READMEs should explain the generic profile/configuration system, update workflow, file layout, safety model, and future customization process.
- Public READMEs should not name exact sensitive profile contexts unless the user explicitly approves those names for publication.

## Plan/final-spec checklist

When planning or creating a config repo, include:

1. A README task that documents how future Hermes custom configuration should be added.
2. A repository-local Git identity task:
   ```bash
   git config user.name "<repo-author-name>"
   git config user.email "<private-term>"
   ```
3. A validation task that rejects AI co-authoring trailers:
   ```bash
   git log --format=%B | grep -Ei 'co-authored-by|ai co-author|generated by ai' && exit 1 || true
   ```
4. A validation task that checks repository-author-authored skill frontmatter:
   ```bash
   python3 - <<'PY'
   from pathlib import Path
   import re, yaml
   for path in Path('.').rglob('SKILL.md'):
       text = path.read_text(encoding='utf-8')
       m = re.search(r'^---\s*\n(.*?)\n---\s*\n', text, re.S)
       if not m:
           raise SystemExit(f'missing frontmatter: {path}')
       fm = yaml.safe_load(m.group(1)) or {}
       if fm.get('author') == '<repo-author-name>':
           assert str(fm.get('version')) == '0.1.0', (path, fm.get('version'))
           assert fm.get('license') == 'Apache-2.0', (path, fm.get('license'))
   print('OK repository-author-authored skill metadata')
   PY
   ```

## Pitfalls

- Do not use `v0.1.0`; the requested skill version string is exactly `0.1.0`.
- Do not treat sensitive profile names as safe for public README examples by default.
- Do not put secrets, token stores, runtime logs, sessions, caches, databases, or profile-private memory in a public config repo.
