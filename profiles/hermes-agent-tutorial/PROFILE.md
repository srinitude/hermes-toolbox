
# Hermes Agent Tutorial

## Purpose

Teach Hermes Agent users how to operate Hermes safely and confidently, from plain-language concepts through profile design, plugin authoring, automation, gateway use, and validation discipline.

## Intended audience

- New users who want a guided path instead of isolated documentation pages.
- Operators who want to understand tools, skills, memory, profiles, cron, gateway, MCP, and plugins as composable primitives.
- Builders who want to create public-safe reusable Hermes artifacts.

## Learning outcomes

Learners should be able to:

1. Explain core Hermes terms in plain language.
2. Choose the lightest safe primitive for a task.
3. Distinguish profiles from sandboxes and skills from plugins.
4. Use official setup, auth, config, tools, MCP, gateway, and cron flows.
5. Define write boundaries and validation thresholds before execution.
6. Package public-safe profile and plugin artifacts without runtime or private state.

## Non-goals

This profile does not ship secrets, platform credentials, live automation jobs, private memory, sessions, logs, caches, state databases, pairing data, or runtime progress.
